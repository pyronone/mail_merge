00 sample cfg

createSampleCfg

 createSampleCfg (_fn:str='sample_cfg.yml', _overwrite:bool=False)

Create sample cfg file in cwd

Type Default Details
_fn str sample_cfg.yml str
_overwrite bool False bool
Returns None

The config file uses YAML format for easy editing. Here you specify the data / template filepaths, as well as which groups require which templates. For example:

data: 'C:\...\tests\example\clean_dat2.parquet'
id_col: 'id'
group_col: '_group'
multi_line_cols:  # optional
  - 'str_mailing_details'
tables:  # optional
  - 'tbl1'
  - 'tbl2'
groups: 
- group1:
  - 'C:\...\tests\example\tpl\sample_tpl.docx'
  - 'C:\...\tests\example\tpl\sample_tpl2.docx'
- group2:
  - 'C:\...\tests\example\tpl\sample_tpl.docx'

This is particularly useful when you have many groups, each requiring different (combinations of) templates.

01 richtext

For variables where the parts of the string should have different styles. Example:

“Parts of a string where there are bolded elements and italicized elements.”

Note: RichText ignores styling and uses the default style in the Word template unless explicitly specified.

Use the patched RT class in this module and initialize with a RTFormatOpts instance to set up default styling options.

An example can be found in the 03 example/richtext section below.

RTFormatOpts

 RTFormatOpts (style:str=None, color:str=None, highlight:str=None,
               size:int=22, subscript:bool=None, superscript:bool=None,
               bold:bool=False, italic:bool=False, underline:str=False,
               strike:bool=False, font:str='Arial', url_id:str=None)

Default options:

style: str = None  # based on style name in Word doc
color: str = None  # e.g., `#00FFFF`
highlight: str = None  # e.g., `#FFFF00`
size: int = 22  # == 11 in Word 
subscript: bool = None   
superscript: bool = None  
bold: bool = False
italic: bool = False
underline: str = False  # options: `'single', 'double', 'thick', 'dotted', 'dash', 'dotDash', 'dotDotDash', 'wave'`
strike: bool = False
font: str = "Arial"
url_id: str = None  # e.g., `url_id=tpl.build_url_id("http://google.com")` where `tpl` is a `DocxTemplate` object

RT

 RT (_default_opts=None, text=None, **text_prop)

Inherits from base docxtpl RichText and adds a few QOL methods. When initializing, pass in RTFormatOpts only.

richtext

def sampleRichText(tr: pd.Series) -> RichText:
    """-> RichText

    Parameters
    ----------
    tr : pd.Series
        pd.Series

    Returns
    -------
    RichText
        RichText
    """
    comment = tr["str_comment"].strip()
    full_name = tr["fname"].strip() + " " + tr["lname"].strip()
    amt_total = tr["amt_total"]

    _opts = RTFormatOpts(italic=True, color="#0000FF")
    rt = RT(_opts)

    rt.addText(comment)
    rt.addText(" ")
    rt.addText(full_name, bold=True)
    rt.addText(" ")
    rt.addText(comment)
    rt.addText(" ")
    rt.addText(amt_total, bold=True)
    rt.addText(".")

    return rt
